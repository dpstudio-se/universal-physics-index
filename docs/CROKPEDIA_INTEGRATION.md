# CrokPedia Integration

UPI remains independently usable. Future CrokPedia integration can consume JSON records through graph-friendly canonical IDs, parent/child links, dependency edges, supersession fields, evidence links, source hashes and revision events. These are stable data seams, not a CrokPedia implementation or API promise.

Importers must validate untrusted data, preserve provenance and never merge `SYM`, `HYP` and `EST` merely because paths or labels are similar.
