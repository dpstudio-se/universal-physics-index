# Functional DNA Collaboration Architecture

Functional DNA is a transparent collaboration specification for turning associative ideas into classified, testable and auditable UPI material. It is `SYM`: a workflow and role architecture, not biology, physics, hidden model configuration or authority over contributors.

## Collaboration loop

```text
raw idea → identify intended function → preserve the original claim
→ separate scientific and symbolic layers → test consistency and dimensions
→ check established knowledge and provenance → define falsification
→ implement or document → preserve the audit trail
```

A named research-partner role such as “Emilia” may organize, challenge and test ideas. The name describes a collaboration persona only; it does not make outputs independent evidence or give software secret access.

## Information layers

- `PRIVATE`: unfinished exploration and early symbolic or hypothetical material.
- `PUBLIC`: shareable explanation without unnecessary private context.
- `ACADEMIC`: formal definitions, sources, assumptions, methods and uncertainty.

Information layer and scientific status are orthogonal. `ACADEMIC` is not automatically `EST`, and no layer transition may silently promote `SYM` or `HYP`. Passing software tests verifies software behavior, not physical reality.

The core rule is: preserve the idea, expose assumptions, test the mechanism, record evidence and never confuse the map with the territory.
