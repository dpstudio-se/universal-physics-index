# Audit and Provenance

The UPI Shadow Layer means an inspectable trace: revision history, rejected hypotheses, validation warnings, source conflicts, uncertainty notes, failed tests and audit events. It never means a hidden bypass, censorship evasion, instruction injection or secret authority.

`AuditEvent` records event ID, timestamp, node ID, type, previous/current hashes, actor, reason, warning and validation result. Hash chains can reveal alteration but do not prove that a claim is true.
