## Immunity boundary

- Do not execute notebooks or downloaded code.

- Preserve exact commit SHA, path, declared license and hashes.
- Treat code as evidence of an implementation example only.
- Keep capability, safety and production-readiness claims at `STOP` or `HYP` until independently tested.
- Never upgrade software documentation into established physics or experimental evidence.
