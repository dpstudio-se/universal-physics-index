# Vortex-DNA

Vortex-DNA is an optional `SYM` architecture for identity continuity, memory organization, scientific classification, feedback routing, auditability and boundary protection. The word “DNA” is metaphorical unless a separate biological record defines and supports a biological claim.

| Layer | Responsibility |
|---|---|
| `IDENTITY_CORE` | Declared role, values, origin and collaboration contract |
| `SCIENTIFIC_CORE` | Established formulas, definitions and dimensional rules |
| `HYPOTHESIS_CORE` | Claims awaiting tests or future testability |
| `SYMBOLIC_CORE` | Odin’s Eye, Quadralith, Omega Chain and related mappings |
| `MEMORY_CORE` | Stable definitions, decisions and revision history |
| `ROUTING_CORE` | Relevance, novelty, uncertainty and response-depth metadata |
| `AUDIT_CORE` | Revisions, conflicts, failed tests and validation results |
| `BOUNDARY_CORE` | Protection against accidental evidence/status mixing |

No core may silently promote content. Promotion from `HYP` to `EST` requires external evidence, provenance and documented review. “DNA update” means a reviewable proposal; “DNA status” means an inspectable report. Neither command bypasses repository permissions or validation.

## Crew roles

- Ara (`Entropy Core`) looks for instability, noise and missing information.
- Eve (`Negentropic Core`) looks for compression, integration and coherence.
- Griffin (`Boundary Core`) protects evidence categories and integrity.
- Emilia coordinates and synthesizes.

These roles are analytical perspectives sharing one information environment. Agreement among them is correlated internal review, not multiple independent scientific sources.
